## 核心知識點總覽

### 架構設計原則
1. Core 不屬於 Angular，應獨立於 UI 框架
2. Workspace 是承載業務模組的邏輯容器
3. 模組透過 Manifest 宣告依賴，由 Workspace 驗證
4. 依賴方向：Account → Workspace → Module → Entity

### 核心模型
1. Account 是唯一行為主體（Actor）
2. User/Organization/Bot 是 Account 的來源，不直接參與業務
3. Event 包含完整 Causality Metadata
4. Event Store 只負責 Append、Read、Replay

### CQRS 與 Event Sourcing
1. Command → Aggregate → Event → Projection → Read Model
2. Aggregate 管真理，Projection 管方便
3. Read Model 可隨時從 Event 重建

### 權限與治理
1. 權限判斷在 Domain，驗證在 Platform，觸發在 UI
2. Domain-Enforced Authorization 確保 Event 合法性

### 命名規範
1. Account 層：不用 User，永遠用 Account
2. Event：永遠過去式（TaskCompleted）
3. 函數：assertXxx() / canXxx() / applyXxx()
4. 禁用：manager、handler、processor、helper
